# GitHubリポジトリ検索アプリケーション 機能仕様書 v1.0

## プロジェクト概要

Next.jsを使用したGitHubリポジトリの検索・閲覧機能を持つWebアプリケーション

### 提供する価値
- 効率的なリポジトリ検索体験
- モダンで直感的なUI
- スムーズな検索・閲覧フロー

## 機能要件

### 基本機能
1. **検索一覧画面**
   - GitHub リポジトリの文字列検索
   - 検索結果の無限スクロール表示
   - リポジトリカードレイアウト

2. **詳細画面**
   - リポジトリの詳細情報表示
   - 検索画面からの遷移

### ユーザーフロー
```
ホーム画面（検索） → 検索実行 → 結果一覧表示 → カードクリック → 詳細画面
```

## 技術要件

### フロントエンド
- **フレームワーク**: Next.js 14 (App Router)
- **UI ライブラリ**: HeroUI（NextUI v2）
- **言語**: TypeScript
- **スタイリング**: Tailwind CSS

### API 連携
- **GitHub REST API**: リポジトリ検索・詳細取得
- **無限スクロール**: GitHub API のページネーション活用

### パフォーマンス要件
- 初回ロード: 3秒以内
- 検索レスポンス: 2秒以内
- 無限スクロール: スムーズな読み込み

## UI・UX 仕様

### デザインシステム
- **テーマ**: モダン・ミニマル
- **カラー**: HeroUI のデフォルトテーマベース
- **レスポンシブ**: モバイルファースト

### 検索一覧画面
#### レイアウト構成
```
+----------------------------------+
|          ヘッダー                 |
+----------------------------------+
|                                  |
|        検索コンポーネント          |
|                                  |
+----------------------------------+
|                                  |
|       検索結果カード一覧          |
|       (無限スクロール)            |
|                                  |
+----------------------------------+
```

#### リポジトリカード仕様
- **表示項目**:
  - リポジトリ名
  - 説明文
  - プログラミング言語
  - スター数
  - フォーク数
  - 最終更新日
- **インタラクション**: ホバー効果、クリック遷移

### 詳細画面
#### 表示項目
- リポジトリの基本情報
- README の内容（optional）
- コントリビューター情報
- Issues・PR 統計
- 戻るボタン

## データ仕様

### GitHub API エンドポイント
1. **検索API**: `GET /search/repositories`
   - パラメータ: `q` (検索クエリ), `page`, `per_page`
   
2. **詳細API**: `GET /repos/{owner}/{repo}`
   - リポジトリの詳細情報取得

### データフロー
```
検索入力 → GitHub API 呼び出し → レスポンス処理 → UI 更新
```
